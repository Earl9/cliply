# Cliply v0.1.0-alpha 手动测试清单

## 范围

本清单只覆盖发布前稳定性测试。不要在本阶段加入云同步、账号、OCR、AI、插件或其他新功能。

## 启动与窗口

- 使用 `npm run tauri -- dev` 启动 Cliply。
- 确认 WebView 加载后主窗口约 2 秒内可见。
- 点击关闭按钮，确认窗口隐藏而不是退出应用。
- 从托盘或全局快捷键重新打开窗口，确认窗口能正常获得焦点。
- 打开日志后确认存在 `window_show duration_ms=...` 记录。

## 剪贴板捕获

- 从记事本复制普通文本，确认左侧出现新的文本记录。
- 连续复制同一段文本，确认启用忽略重复时复用已有记录并更新排序/次数。
- 复制 URL，确认记录类型为链接。
- 复制代码片段，确认记录类型为代码。
- 复制图片，确认图片缩略图和详情预览都能加载。
- 暂停监听后复制内容，确认不会新增记录。
- 恢复监听后复制内容，确认记录再次正常新增。

## 错误提示

- 尽量用其他程序占用剪贴板，再触发复制，确认出现“剪贴板读取失败，请稍后重试”。
- 临时让应用数据目录不可写，确认写入失败时出现“数据库写入失败”相关提示。
- 在无法粘贴或无焦点目标中执行粘贴，确认出现“粘贴失败，内容已尽量复制到剪贴板”。
- 删除或重命名某条图片记录对应的图片文件，确认详情区显示“图片加载失败”或“图片文件不可用”。

## 数据操作

- 固定一条记录，刷新或重启后确认固定状态仍存在。
- 删除一条记录，刷新或重启后确认记录不再出现。
- 清空历史，确认固定记录保留。
- 分别搜索正文、URL、来源应用和标签，确认结果实时过滤。
- 分别切换全部、文本、链接、图片、代码、固定筛选，确认列表真实变化。
- 使用 ArrowUp / ArrowDown 移动选中项，确认右侧详情同步变化。

## 日志

- 打开 About -> Debug，找到日志文件路径。
- 确认 `cliply.log` 包含启动、存储初始化、监听启动、捕获、重复项、窗口显示和错误事件。
- 确认日志只包含事件名、记录 ID、类型和错误信息，不包含剪贴板正文。

## SQLite Limit

- 确认前端列表查询传入 `limit`。
- 确认后端将列表查询 `limit` 限制在 1 到 200 之间。
- 确认搜索查询同样带有 `LIMIT` 和 `OFFSET`。

## 已知 Alpha 风险

- Windows 剪贴板被其他应用占用时，读取可能短暂失败。
- 粘贴模拟依赖当前前台应用焦点，部分高权限窗口可能拒绝粘贴。
- 图片格式支持仍是 MVP 范围，少数剪贴板图片变体可能无法解析。
